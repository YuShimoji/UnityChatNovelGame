# コンセプト補助メモ（GDDの要約）

更新日: 2026-02-10

このファイルは、散在しているメモや仕様を要約した補助メモ。
**SSOT は `docs/GAME_DESIGN_DOCUMENT.md`** とし、矛盾が出た場合は GDD を優先する。

**メタプロンプト（仕様策定の進め方）**
次に仕様を追加・更新するときは、以下に沿って「判断の軸」を先に書き、後から詳細を書く。

1. 目的: その仕様でプレイヤー体験の何を改善するか。
2. 成果物: 画面・データ・フローのいずれが確定するか。
3. 制約: 技術的・制作的に守るべき制約は何か。
4. 依存: Yarn変数、Topic、UI、音/演出など他システムへの影響。
5. 受け入れ条件: 何ができれば「仕様完了」と言えるか。
6. MVP優先度: 今すぐ必要か、後回しでよいか。

**ターゲット体験**
ジャンル: Found Phone スタイルのチャットノベル（拾った設定に限定しない）
プレイヤーの役割: スマホ内の会話を通じて状況に関与する当事者
体験の核: 没入、会話の手触り、選択による分岐

**デザイン・ピラー**
1. 「手がかりロス」を排除する。トピックは消費ではなく「解放（Unlock）」として扱う。
2. 物語は**手作り**で進行し、自動生成は行わない。
3. 画面外の安全圏を侵食するメタ演出で没入を作る。ただし不快感は最小化する。
4. チャットアプリとして自然な手触りを守る（テンポ、通知、入力体験）。

**エンジン方針**
チャットノベルゲームエンジンとしてのコア機能を最優先する。
物語テーマ（メタホラー/FoundPhone 等）は分離可能とし、エンジンは共通基盤として保つ。
サンプルストーリーの要求から標準機能を逆進的に追加することを許容する。

**コアゲームループ**
1. チャットで物語が進行する。
2. 必要に応じてトピック（手がかり）を獲得する。
3. トピックの提示・選択で分岐が開く。

**主要システム（統合方針）**
チャットシステム  
UIはScrollRect + バブル。自動スクロールは「過去ログ閲覧時は抑制」。

シナリオ基盤  
Yarn Spinnerで分岐と進行を制御し、カスタムコマンドでUIや演出を呼び出す。

待機とテンポ  
リアルタイム連動は行わず、最長15秒の待機。行動（ミニゲーム等）で短縮できる。

探索スレッド  
メインと独立した「クエスト」だが、取得トピックがメイン進行の燃料になる。

メタホラー演出  
グリッチ、ゴーストタイピング、UIの違和感演出。実デバイス機能の実利用はしない。

**ナラティブ構造**
進行はチャット主導。メインとサブの境界を薄くし、選択と提示で展開が変わる。
物語は手作りであり、ミステリーの自動生成は行わない。

**演出と没入の基準**
スマホが「呪われた/乗っ取られた」ように錯覚させる。
第四の壁演出は強力だが、やりすぎは不快につながるため段階制で制御する。
カメラ起動は演出上のフェイクのみ。実際のアクセスは行わない。

**技術・制作制約**
リアルタイム連動なし。待機は最大15秒。
アセット最小化。フォントアイコンやシンプル素材を優先。
Safe Areaとキーボード対応は必須。
データはScriptableObjectとYarn変数の橋渡しで管理する。

**MVP定義**
チャットUI、Yarn連携、カスタムコマンド。
待機システム（Short Waiting）と簡易ミニゲーム1種。

**非ゴール（今はやらない）**
長時間のリアルタイム待機。
高負荷な3D演出や重い長編ミニゲーム。
実端末機能の侵襲的利用（カメラ/マイク/ストレージ等の実アクセス）。

**未決事項（要判断）**
トピック提示・選択の具体仕様。
探索スレッドの失敗時リワード設計。
メタホラー演出の強度レベルと解除条件。
メインストーリーの章数と各章のゴール。
